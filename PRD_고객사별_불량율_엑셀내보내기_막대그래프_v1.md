# PRD: 고객사별 불량율 엑셀 내보내기 + 엑셀 기반 막대 그래프
작성일: 2026-04-21  
대상: [index.html](c:\Users\Public\부적합\shinsung-ncp-management-main\index.html)

## 1. 배경
현재 대시보드는 `주요3사 고객불만 현황` 탭에서 불량율 추이(꺾은선), 표, 유형 도넛을 제공한다.  
다만 다음 니즈가 남아 있다.

- 현재 화면 기준 결과를 엑셀 파일로 직접 내보내고 싶다.
- 엑셀 파일(내보낸 파일 또는 외부 집계 파일) 기준으로 고객사별 불량율을 막대 그래프로 보고 싶다.

## 2. 목표
- `index.html` 내에서 고객사별 불량율 데이터를 `.xlsx`로 내보내는 기능을 제공한다.
- 업로드한 엑셀 데이터로 고객사별 불량율 막대 그래프를 렌더링한다.
- 기존 `tab-defect-rate` 흐름(기간 필터, 수량/건수 모드, 목표불량율, 인쇄)은 유지한다.

## 3. 범위
### 포함 범위 (P0)
- `tab-defect-rate`에 `고객사 불량율 XLSX 내보내기` 버튼 추가.
- `tab-defect-rate`에 `XLSX 불러오기(막대그래프)` 버튼 추가.
- 고객사별 불량율 집계 시트 생성 및 다운로드.
- 업로드한 집계 시트를 파싱해 고객사별 불량율 막대 그래프 렌더링.
- 데이터/형식 오류 시 사용자 메시지 제공.

### 제외 범위 (Out of Scope)
- 서버 업로드/공유 저장.
- 엑셀 파일 내부에 차트 오브젝트 삽입.
- 다중 파일 병합 분석.

## 4. 사용자 시나리오
1. 사용자는 `주요3사 고객불만 현황` 탭에서 연도/기간/모드를 선택한다.
2. `고객사 불량율 XLSX 내보내기` 버튼을 눌러 집계 파일을 받는다.
3. `XLSX 불러오기(막대그래프)` 버튼으로 파일을 다시 열거나, 외부에서 만든 동일 포맷 파일을 연다.
4. 고객사별 불량율 막대 그래프를 확인한다.

## 5. 기능 요구사항
### FR-01. 엑셀 내보내기
- 버튼 클릭 시 현재 필터 기준으로 고객사별 집계를 계산한다.
- 파일명 규칙: `신성텍_고객사별_불량율_<YYYYMMDD_HHMM>.xlsx`
- 시트 구성:
- `집계` 시트: 그래프용 고객사 집계 데이터
- `원본(선택)` 시트: 집계 근거 행 데이터 (P1)
- `메타` 시트: 생성일시, 기준기간, 모드, 파일명

### FR-02. 집계 컬럼 정의 (`집계` 시트)
- `고객사`
- `입고수량`
- `검사수량`
- `입고건수`
- `부적합수량`
- `부적합건수`
- `불량율_수량(%)` = `부적합수량 / 입고수량 * 100`
- `불량율_건수(%)` = `부적합건수 / 입고건수 * 100`
- `표시불량율(%)` = 화면 모드가 `수량`이면 `불량율_수량(%)`, `건수`면 `불량율_건수(%)`

### FR-03. 엑셀 불러오기
- 입력 파일에서 우선 `집계` 시트를 찾고 없으면 첫 시트를 사용한다.
- 필수 컬럼:
- `고객사`
- `표시불량율(%)` 또는 `불량율_수량(%)`/`불량율_건수(%)` 중 1개 이상
- 숫자 파싱 실패 행은 제외하고 경고 건수를 표시한다.

### FR-04. 막대 그래프 렌더링
- 그래프 ID: `c-dr-vendor-bar` (신규)
- X축: 고객사, Y축: 표시불량율(%)
- 정렬: 불량율 내림차순
- Hover: 고객사, 표시불량율, 수량/건수 분모·분자
- 목표불량율이 있을 경우:
- 목표 초과 막대 색상 강조
- 목표선(수평 점선) 표시

### FR-05. 데이터 소스 전환
- 소스 상태: `live`(현재 대시보드 계산값) / `excel`(업로드 집계값)
- 기본값: `live`
- `excel` 소스가 없으면 자동 `live` 사용

## 6. UI 요구사항
- 위치: `tab-defect-rate` 상단 필터 영역 우측.
- 신규 UI 요소:
- 버튼 `📥 고객사 불량율 XLSX`
- 버튼 `📊 XLSX 불러오기`
- 숨김 파일 입력 `input[type=file]` (`.xlsx,.xls,.xlsm`)
- 그래프 박스 `고객사별 불량율(막대)`
- 기존 UI 톤(`btn`, `chart-box`, `cm-year-select`)과 동일 스타일 사용.

## 7. 기술 설계
### 7.1 재사용
- 엑셀 처리: 기존 `XLSX` 라이브러리 재사용.
- 차트: 기존 `Plotly.react` 재사용.
- 유틸: `fileStamp`, `fmt`, `num`, `downloadBlob` 재사용.

### 7.2 신규 함수(안)
- `buildDrVendorAggregateRows({year, monthStart, monthEnd, mode})`
- `exportDefectRateWorkbook()`
- `parseDefectRateWorkbook(file)`
- `renderDefectRateVendorBar({source})`
- `getDefectRateBarColor(rate, target)`

### 7.3 상태(안)
- `G.drVendorRate = { source: 'live'|'excel', rows: [] }`
- 필요 시 `localStorage` 키: `dr_vendor_rate_source`

## 8. 예외/오류 처리
- 집계 대상 고객사가 0건이면 내보내기 차단 후 안내.
- 파일 파싱 실패 시 `alert` 대신 기존 에러 배너/토스트 방식 사용.
- 필수 컬럼 누락 시 누락 컬럼명을 포함한 안내 메시지 제공.

## 9. 수용 기준 (Acceptance Criteria)
- AC-01: `tab-defect-rate`에서 버튼 클릭 시 `.xlsx`가 정상 다운로드된다.
- AC-02: 다운로드 파일 `집계` 시트에 고객사별 불량율 컬럼이 포함된다.
- AC-03: 업로드한 파일 기준으로 막대 그래프가 렌더링된다.
- AC-04: 목표불량율 설정 시 목표선과 초과 강조가 적용된다.
- AC-05: 잘못된 파일 업로드 시 앱이 중단되지 않고 오류 메시지를 보여준다.
- AC-06: 기존 꺾은선/표/도넛 렌더링 동작은 회귀 없이 유지된다.

## 10. 테스트 시나리오
- TS-01: 정상 데이터 내보내기 후 재업로드 시 그래프 값 일치 확인.
- TS-02: `수량/건수` 모드 전환 후 내보내기 값(`표시불량율`) 변경 확인.
- TS-03: 고객사 1개/다수/0건 케이스 확인.
- TS-04: 목표불량율 미입력/입력 상태에서 그래프 스타일 확인.
- TS-05: 필수 컬럼 누락 엑셀 업로드 시 에러 메시지 확인.

## 11. 구현 단계
- 1단계(P0): 내보내기 + 업로드 파싱 + 막대 그래프 렌더링.
- 2단계(P1): `원본` 시트 추가, Top N 필터(10/20/전체), 소스 상태 영구 저장.
- 3단계(P2): 엑셀 템플릿 검증 강화(헤더 매핑 규칙 확장).

## 12. 리스크 및 대응
- 리스크: 업로드 파일 포맷 다양성으로 파싱 실패 가능.
- 대응: 필수 컬럼 최소화 + 컬럼명 별칭 매핑 + 상세 에러 메시지.
- 리스크: 기존 탭 성능 저하.
- 대응: 집계 결과 캐시 및 `renderDefectRate`와 분리 렌더.
