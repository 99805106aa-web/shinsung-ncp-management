# 신성텍 NCP 관리 시스템 — 기능 확장 PRD
**Product Requirements Document v1.0**
작성일: 2026-02-24 | 대상 시스템: `index.html` (단일 파일 웹 앱)

---

## 1. 배경 및 현황

### 현재 시스템이 하는 일
| 영역 | 현재 기능 |
|------|-----------|
| 데이터 입력 | Excel 파일 업로드 / 드래그앤드롭 |
| KPI 요약 | 검사수량 · 불량수량 · 불량율 · 사외건수 · 업체수 |
| 사내 분석 | 거래처별·유형별·일별 차트, 품번 Top10, 리스크 요약 |
| 사외 분석 | 부적합 유형·품목별 차트, 상세 테이블, 첨부파일 관리 |
| AI 분석 | 인사이트 텍스트, 리스크 경보, 재발율 계산 |
| 라이브 동기화 | Excel 변경 시 자동 갱신 |

### 현재 시스템의 한계
- **사후 집계 중심**: 이미 발생한 불량을 보여줄 뿐, 예측·예방 분석 없음
- **개선 대책 추적 불가**: 조치내용이 텍스트 컬럼 하나뿐, 완료 여부 추적 불가
- **고객사 성과 평가 없음**: 납품받은 고객사별 품질 등급이 없음
- **통계 기법 부재**: SPC, 공정능력, 파레토 등 제조업 핵심 기법 미적용
- **비교 분석 없음**: 전월/전년 동기 대비 데이터 없음
- **모바일 미최적화**: 태블릿·현장 단말에서 사용 어려움

## 2. 목표 및 우선순위

> 사용 편의성(현장 작업자) + 분석 깊이(품질 관리자) + 예방 중심(경영진) 세 관점을 균형 있게 확장

### 우선순위 기준
- **P0**: 즉각적 업무 효율 향상, 구현 복잡도 낮음
- **P1**: 품질 관리 핵심 지표, 중간 복잡도
- **P2**: 고급 분석 기능, 높은 복잡도 또는 데이터 추가 필요

### F-01 · 파레토 분석 탭 (Pareto Analysis)
**우선순위: P0** | 예상 공수: 2일

#### 배경
품질 관리의 기본 원칙인 "80/20 법칙" — 불량의 80%는 20%의 원인에서 발생. 현재 시스템엔 누적 비율 시각화가 없어 집중 개선 대상을 빠르게 파악하기 어렵다.

#### 요구사항
- **파레토 차트**: 불량 유형 / 거래처 / 품번을 X축으로, 수량(막대) + 누적 비율(꺾은선, 2차 Y축) 동시 표시
- **분석 기준 선택**: 불량수량 / 불량율 / 건수 중 선택
- **대상 선택**: 사내불량 / 사외불량 / 통합 중 선택
- **80% 임계선**: 누적 80% 지점에 수직 점선 표시 + "이 구간 이하 개선 시 불량의 80% 감소" 텍스트
- **드릴다운**: 막대 클릭 시 해당 유형/거래처/품번으로 필터 적용

#### 구현 방식
- 기존 Plotly 라이브러리 활용 (추가 의존성 없음)
- `tab-chart` 내 새 섹션으로 추가하거나 독립 탭으로 분리

#### 성공 지표
- 클릭 3번 이내에 "핵심 개선 대상 Top 3" 파악 가능

### F-02 · 기간 비교 분석 (Period Comparison)
**우선순위: P0** | 예상 공수: 3일

#### 배경
"이번 달이 지난달보다 나쁜가?"를 즉시 알 수 없다. 헤더의 기간 표시만 있고 전기 대비 비교 수치가 없어 경영진 보고에 약점.

#### 요구사항
- **비교 기간 선택 UI**: "이번 달 vs 지난 달", "올해 vs 작년 동기", "사용자 지정 구간 A vs B"
- **비교 KPI 카드**: 각 KPI에 ▲▼ 변화량 및 % 증감 표시 (빨강/초록 색상)
- **오버레이 차트**: 일별 추이 차트에 비교 기간 데이터를 반투명 선으로 오버레이
- **요약 텍스트**: "불량율이 전월 대비 0.42%p 악화, 외관불량이 주요 원인" 형태 자동 생성

#### 구현 방식
- 현재 `G.saenae` / `G.saewae` 데이터를 날짜 기준으로 양분하여 두 집합 비교
- 날짜 필터 UI에 "비교 모드" 토글 버튼 추가

#### 성공 지표
- 전월 대비 불량율 증감을 로딩 후 5초 이내 확인 가능

### F-03 · 고객사 성과 스코어카드 (Customer Scorecard)
**우선순위: P1** | 예상 공수: 3~4일

#### 배경
신성텍은 부품 공급사이며, 사외불량은 납품받은 **고객사**가 제기한 부적합이다.
현재 고객사별 불량수량 차트는 있지만 종합 품질 등급이 없어 "어느 고객사에서 클레임이 집중되는가"를 한눈에 알 수 없다.

#### 요구사항
- **등급 산출 로직** (100점 만점):
  - 부적합율 (40점): 낮을수록 고점
  - 재발율 (30점): 사외 재발 건수 기반
  - 개선대책서 제출율 (20점): 첨부 완료 비율
  - 부적합 추세 (10점): 최근 3개월 부적합율 개선 방향
- **등급 구분**: A(90↑) / B(75↑) / C(60↑) / D(60미만)
- **스코어카드 테이블**: 고객사명, 총 출하수량, 부적합율, 재발율, 첨부율, 종합점수, 등급, 추세 아이콘
- **레이더 차트**: 고객사 선택 시 4개 항목 레이더 차트 팝업
- **등급 배지**: 사외불량 테이블의 고객사명 옆에 등급 배지 표시

#### 구현 방식
- 새 탭 "🏢 고객사 평가" 추가
- 점수 계산은 `buildCustomerScorecard(sw)` 함수로 순수 JS 구현

#### 성공 지표
- 품질 관리자가 고객사별 클레임 등급을 즉시 확인하고 집중 대응 가능

### F-04 · 관리도 (SPC Control Chart)
**우선순위: P1** | 예상 공수: 4~5일

#### 배경
제조업 품질 관리의 핵심인 SPC(Statistical Process Control). 불량율이 "관리 한계선"을 벗어나는 이상징후를 조기 감지해야 한다.

#### 요구사항
- **P 관리도 (불량율 관리도)**:
  - 중심선 (CL): 평균 불량율 p̄
  - 상한 관리한계 (UCL): p̄ + 3√(p̄(1-p̄)/n̄)
  - 하한 관리한계 (LCL): max(0, p̄ - 3√(p̄(1-p̄)/n̄))
  - 관리 이탈 포인트: 빨간 점 + 경고 아이콘
- **이상 판정 규칙** (Western Electric Rules 일부):
  - 규칙 1: UCL 초과 또는 LCL 미달 1점
  - 규칙 2: 연속 8점이 중심선 한쪽에 집중
  - 규칙 3: 연속 6점 단조 증가/감소
- **분석 대상 선택**: 품번별 / 거래처별 / 전체
- **이상 판정 시 경고**: 빨간 배너 + "2026-02-10 UCL 초과 — 즉각 점검 필요" 텍스트

#### 구현 방식
- Plotly shapes로 UCL/LCL 수평선 표시
- 기존 `renderChartDaily` 확장 또는 새 함수 `renderChartSPC` 추가

#### 성공 지표
- 공정 이상 발생 시 차트에서 자동으로 시각적 경고 확인 가능

### F-05 · 개선 대책 추적 (Corrective Action Tracker)
**우선순위: P1** | 예상 공수: 4~5일

#### 배경
현재 "조치내용"은 텍스트 1줄뿐, 개선이 완료됐는지 미완인지 추적 불가. 사외 부적합 개선대책서도 첨부 여부만 알 수 있고 내용/진행 상태 파악 불가.

#### 요구사항
- **상태 컬럼 추가**: 각 불량 행에 `미조치 / 진행중 / 완료 / 재발` 상태 선택 가능
- **KPI 카드**: "미조치 건수 N건", "30일 이상 미완료 건수 N건" 추가
- **에이징 분석**: 불량 발생일 기준 경과일 계산, 30일↑ 노란색, 60일↑ 빨간색
- **마감일 설정**: 각 행에 목표 완료일 입력 필드
- **필터**: 상태별 / 에이징 구간별 필터
- **상태 데이터 저장**: `localStorage`에 JSON으로 저장 (Excel 파일과 별개 관리)

#### 구현 방식
- 사외불량 테이블에 상태 컬럼 + 상태 배지 추가
- 상태 변경은 드롭다운으로 인라인 편집
- `localStorage` 키: `ncp_action_status_v1` (품번+날짜+업체로 복합키)

#### 성공 지표
- "지금 미조치 건이 몇 건인가?"를 1초 이내 확인 가능

### F-06 · 불량 예측 트렌드 (Defect Trend Forecast)
**우선순위: P1** | 예상 공수: 3일

#### 배경
과거 추이를 보면 다음 달 불량을 예측할 수 있다. 현재 추이 차트는 과거만 보여주고 미래 예측이 없다.

#### 요구사항
- **선형 회귀 예측선**: 최근 30일 데이터를 기반으로 향후 14일 예측값 점선으로 표시
- **신뢰 구간**: 예측값 ±σ 범위를 반투명 영역으로 표시
- **예측 텍스트**: "현 추세 유지 시 다음 달 불량율 예상: 2.3% (±0.5%)"
- **예측 정확도 표시**: 이전 달 예측값과 실제값 비교하여 모델 신뢰도 표시
- **경고 예측**: 예측값이 목표 불량율 초과 시 "불량율 목표 초과 예상" 경고 배너

#### 구현 방식
- 외부 라이브러리 없이 최소자승법(Least Squares) 순수 JS 구현
- 기존 일별 추이 차트에 예측 레이어 추가 (토글 버튼으로 ON/OFF)

#### 성공 지표
- 다음 달 불량율 예측값을 현재 데이터에서 자동 계산하여 표시

### F-07 · 보고서 PDF/Excel 내보내기 (Export)
**우선순위: P1** | 예상 공수: 3일

#### 배경
현재 경영진 보고를 위해 화면 캡처를 수동으로 하고 있을 가능성이 높다. 클릭 한 번에 보고서를 생성하면 업무 시간 대폭 절약.

#### 요구사항
- **Excel 내보내기**: 현재 필터 상태의 사내/사외 데이터를 .xlsx로 다운로드
  - 시트 1: KPI 요약
  - 시트 2: 사내 상세
  - 시트 3: 사외 상세
  - 시트 4: 품번별 집계
- **HTML 인쇄 최적화**: "인쇄" 버튼 클릭 시 차트+테이블이 A4 레이아웃으로 정렬된 인쇄 뷰 제공 (CSS `@media print` 활용)
- **CSV 내보내기**: 개별 테이블마다 CSV 다운로드 버튼

#### 구현 방식
- 기존 탑재된 `SheetJS(XLSX)` 라이브러리로 쓰기 기능 활용 (`XLSX.write`)
- PDF는 브라우저 기본 인쇄 기능 위임 (외부 라이브러리 불필요)

#### 성공 지표
- 월간 품질 보고서용 Excel을 버튼 1회 클릭으로 생성 가능

### F-08 · 불량 히트맵 (Defect Heatmap)
**우선순위: P2** | 예상 공수: 2일

#### 배경
"어느 요일/주에 불량이 집중되는가?" 패턴을 발견하면 작업 스케줄 조정으로 예방 가능.

#### 요구사항
- **요일 × 주차 히트맵**: X축=요일(월~금), Y축=주차, 셀 색상=불량수량 (연두~빨강 스펙트럼)
- **월 × 연도 히트맵**: X축=월, Y축=연도 (GitHub 잔디처럼)
- **툴팁**: 셀 hover 시 해당 날짜의 주요 불량 정보 표시
- **분석 기준 선택**: 불량수량 / 불량율 / 건수

#### 구현 방식
- Plotly heatmap (z=수량 2D 배열) 활용
- `renderChartHeatmap()` 신규 함수

#### 성공 지표
- 특정 요일/시기에 불량이 집중되는 패턴을 시각적으로 확인 가능

### F-09 · 직접 데이터 입력 폼 (Web-based Entry)
**우선순위: P2** | 예상 공수: 5~7일

#### 배경
현장 작업자가 Excel에 입력 후 시스템에 업로드하는 2단계 프로세스가 있다. 웹에서 직접 입력하면 실시간 반영 가능.

#### 요구사항
- **입력 폼**: 검사일자, 거래처, 품번, 품명, 발주/검사/불량수량, 불량유형(체크박스), 조치내용
- **자동완성**: 기존 데이터에서 품번·품명·거래처 자동완성
- **불량율 실시간 계산**: 검사/불량수량 입력 시 즉시 계산하여 표시
- **입력 후 처리**: `localStorage`에 저장, 업로드된 Excel 데이터와 병합하여 표시
- **Excel 동기화**: 입력된 데이터를 Excel 형식으로 내보내기 (기존 파일에 추가 가이드 제공)

#### 제약 조건
- 순수 프론트엔드 구현 (서버 없음) → `localStorage` + IndexedDB 활용
- 입력 데이터와 Excel 데이터의 중복 방지 로직 필요

#### 성공 지표
- 현장에서 태블릿으로 불량을 바로 입력하고 즉시 차트에 반영 가능

---

### F-10 · 알림 시스템 (Browser Notification)
**우선순위: P2** | 예상 공수: 2일

#### 배경
라이브 동기화로 Excel이 자동 갱신되지만 관리자가 화면을 보고 있지 않으면 중요 이상 발생을 놓친다.

#### 요구사항
- **브라우저 알림 (Web Notification API)**:
  - 사외 부적합 신규 행 발생 시
  - 불량율이 UCL 초과 시
  - 30일 이상 미조치 건 발생 시
- **화면 내 알림 토스트**: 우하단에 슬라이드인 알림 (자동 닫힘 5초)
- **알림 이력**: "🔔 알림 센터" 아이콘 클릭 시 최근 알림 목록 표시
- **설정**: 알림 유형별 ON/OFF 토글

#### 구현 방식
- `Notification.requestPermission()` + `new Notification()`
- 기존 `startLiveSyncWatchers()` 데이터 비교 로직에 알림 훅 추가

## 4. 구현 우선순위 로드맵

```
즉시 적용 가능 (1~2주)

├── F-07 Excel/인쇄 내보내기 ← SheetJS 이미 탑재
└── F-08 불량 히트맵        ← Plotly heatmap 신규 활용

중기 (1~2개월)

├── F-06 불량 예측 트렌드   ← 선형 회귀 구현

장기 (2개월+)
└── F-09 직접 데이터 입력   ← 가장 큰 UX/아키텍처 변화
```
