# PRD: 대표자 월별보고 탭

- 작성일: 2026-02-26
- 버전: v1.0 (초안)
- 대상 화면: `index.html`
- 목적: 대표자가 월마다 같은 형식으로 빠르게 보고받을 수 있게 한다.

## 1. 문서 목적

이 문서는 `월별보고` 탭을 새로 만들기 위한 요구사항 문서다.  
핵심은 어렵지 않은 말로, 한 화면에서 월별 상황을 바로 이해하게 하는 것이다.

## 2. 배경

현재 화면은 데이터와 차트가 많아서, 대표자 보고용으로 핵심만 빠르게 보기 어렵다.

문제는 아래와 같다.

1. 월별 요약이 한 곳에 모여 있지 않다.
2. 같은 형식으로 월별 비교가 어렵다.
3. 설명 문구에 어려운 말이 섞이면 이해 속도가 느려진다.

## 3. 목표

1. `월별보고` 탭 하나로 월별 핵심 내용을 본다.
2. 각 월 탭은 항상 같은 4개 영역으로 보여준다.
3. 문장은 누구나 이해할 수 있는 쉬운 표현만 쓴다.

## 4. 비목표

1. 새 통계 모델 추가는 이번 범위에서 제외한다.
2. 기존 전체 탭 구조를 크게 바꾸지 않는다.
3. 서버 구조 변경은 최소화한다.

## 5. 사용자

1. 대표자: 1~3분 안에 월별 상태 파악
2. 품질팀: 보고 전 확인, 설명 문구 점검

## 6. 화면 구조

### 6.1 상위 탭 추가

- 기존 탭 옆에 `📅 월별보고` 탭 추가
- 탭 ID: `tab-monthly-report`

### 6.2 월 하위 탭(또는 월 선택 버튼)

- 최근 월이 기본 선택
- 최근 12개월 표시 (예: `2026-02`, `2026-01` ...)
- 월을 바꾸면 아래 4개 영역이 모두 같은 월 기준으로 바뀜

## 7. 월별 탭 공통 구성 (핵심)

각 월 탭에는 아래 4개를 반드시 같은 순서로 보여준다.

### 7.1 분석결과

- 형식: 한 줄 요약 3개
- 예시:
  - `이번 달 불량수량은 지난달보다 12% 줄었습니다.`
  - `사외 부적합은 3건으로 지난달과 같습니다.`
  - `미조치 건수는 5건이며, 이 중 2건은 30일이 넘었습니다.`

표시 규칙:

1. 좋은 변화는 파란색/초록색
2. 나쁜 변화는 빨간색
3. 숫자는 정수 또는 소수점 1자리까지

### 7.2 핵심지표

- 형식: KPI 카드 6개
- 지표 목록(고정):
  1. 총 검사수량
  2. 총 불량수량
  3. 불량률(%)
  4. 사외 부적합 건수
  5. 미조치 건수
  6. 30일 이상 미완료 건수

표시 규칙:

1. 각 카드에 전월 대비 증감(+/-)을 작은 배지로 표시
2. 수량은 `건/개`, 비율은 `%`
3. 값이 없으면 `데이터 없음` 표시

### 7.3 핵심키워드

- 형식: 키워드 태그 5개
- 추출 기준:
  1. 불량유형 상위 2개
  2. 영향이 큰 품번 상위 2개
  3. 조치 상태 키워드 1개 (`미조치`, `진행중`, `완료` 중 가장 많은 값)

표시 규칙:

1. 각 태그에 건수 함께 표시 (예: `스크래치 18건`)
2. 어려운 약어는 사용하지 않음

### 7.4 핵심 인사이트

- 형식: 4문장 고정 템플릿
- 문장 틀:
  1. `지금 상태:` 이번 달 전체 상태 한 줄
  2. `주요 원인:` 가장 큰 원인 한 줄
  3. `위험 신호:` 바로 챙겨야 할 내용 한 줄
  4. `다음 행동:` 다음 달 전까지 할 일 한 줄

문장 규칙:

1. 한 문장 30자 내외 권장
2. 어려운 말 금지
3. 숫자 근거 1개 이상 포함

## 8. 쉬운 말 기준 (필수)

아래 말은 바꿔 쓴다.

1. `트렌드` -> `흐름`
2. `인사이트` -> `핵심 내용`
3. `리스크` -> `위험`
4. `액션 아이템` -> `할 일`
5. `증감률` -> `늘고 줄어든 비율`

문구는 품질팀이 수정 가능하게 텍스트 함수로 분리한다.

## 9. 기능 요구사항

1. `FR-01` 월 목록 자동 생성: 데이터의 `날짜(YYYY-MM)` 기준으로 월 목록 생성
2. `FR-02` 기본 월 선택: 가장 최근 월 자동 선택
3. `FR-03` 월 변경 연동: 분석결과/핵심지표/핵심키워드/핵심 인사이트 동시 갱신
4. `FR-04` 빈 데이터 처리: 해당 월 데이터가 없으면 빈 상태 화면 표시
5. `FR-05` 내보내기: 현재 선택 월 내용을 한 페이지 PDF 또는 인쇄 화면으로 제공
6. `FR-06` 성능: 월 전환 후 2초 안에 화면 갱신

## 10. 데이터 규칙

1. 사내/사외 데이터를 월 단위로 합쳐서 계산
2. 기준 월 키: `날짜`의 앞 7자리 (`YYYY-MM`)
3. 비율 계산 시 0으로 나누기 방지
4. 소수점은 최대 1자리
5. 월 비교 기준은 바로 이전 월

## 11. 수용 기준 (Acceptance Criteria)

1. 최근 12개월 월 탭이 정상 표시된다.
2. 월 탭을 바꾸면 4개 영역이 같은 월 데이터로 바뀐다.
3. 핵심지표 6개가 모두 값 또는 `데이터 없음`으로 표시된다.
4. 핵심키워드가 최대 5개 표시되고 건수가 함께 나온다.
5. 핵심 인사이트가 4문장 템플릿으로 출력된다.
6. 어려운 단어 치환 규칙이 실제 문구에 반영된다.

## 12. 개발 순서 제안

1. 1단계: 탭 UI + 월 선택 + 월 필터 로직
2. 2단계: 핵심지표 카드 6개 + 전월 대비 배지
3. 3단계: 분석결과/키워드/핵심 인사이트 자동 문구
4. 4단계: 인쇄/PDF 내보내기 + 문구 다듬기

## 13. 보고용 샘플 출력 형식

- 분석결과
  - 이번 달 불량수량: 128개 (지난달 146개, 18개 감소)
  - 사외 부적합: 4건 (지난달 6건, 2건 감소)
  - 미조치: 7건 (30일 이상 3건)
- 핵심지표
  - 총 검사수량 12,400개 / 총 불량수량 128개 / 불량률 1.0%
  - 사외 부적합 4건 / 미조치 7건 / 30일 이상 미완료 3건
- 핵심키워드
  - 찍힘 22건, 스크래치 18건, P/N-A102 14건, P/N-B221 11건, 미조치 7건
- 핵심 인사이트
  - 지금 상태: 지난달보다 전체 불량이 줄었습니다.
  - 주요 원인: 찍힘과 스크래치가 절반 이상입니다.
  - 위험 신호: 30일 이상 미완료가 3건 남아 있습니다.
  - 다음 행동: 상위 2개 불량유형을 먼저 줄이는 조치를 진행합니다.
